## FocusGuard 项目全景说明书

### 1. 产品定位与核心理念

FocusGuard 是一款面向 Windows 平台的 **极简、高阻力、智能化专注工具**。  
它通过“事前承诺的番茄钟 + 多层级拦截引擎 + 大模型智能护航”，在不打扰正常工作的前提下，**主动阻断娱乐与摸鱼入口**，帮助用户长时间保持深度专注：

- **极简**：常驻托盘、无冗余弹窗，所有核心操作集中在“控制中心”与少量托盘菜单中完成。
- **高阻力**：一旦进入专注周期，程序会“焊死退路”：禁止常规退出路径，仅保留带有 60 秒冷静期的紧急安全阀。
- **智能化**：在传统进程/标题黑名单之上，引入浏览器扩展与云端 LLM，对网页意图做语义级判断，**奖励学习型内容、拦截纯娱乐内容**。

### 2. 当前架构图解（文本版）

#### 2.1 核心后台（守护进程与阻断引擎）

- `main.py`：程序主入口
  - 启动并持有全局状态控制器 `FocusState`：
    - 管理专注模式开关、专注结束时间 `end_time`。
    - 管理悬浮倒计时子进程与控制中心子进程生命周期。
    - 暴露 `focus_target` 字段，记录当前专注目标文本。
  - 启动后台线程 `monitor_loop`：
    - 每 1.5 秒轮询前台窗口信息（通过 `monitor.get_active_window_info()`）。
    - 读取并消费 `focus_command.json`，触发新一轮专注（包含时长与目标）。
    - 在专注模式下，根据前台是否为浏览器进程，从 `server.CURRENT_BROWSER_TAB` 获取 `browser_title/browser_url`，并调用 `blocker.enforce_rules(...)`。
    - 专注到点自动弹出系统原生完成提示框。
  - 启动本地 HTTP 微服务（`server.start_server`）与控制中心子进程（`dashboard_ui.py`），再进入托盘 UI 主循环。

- `monitor.py`：前台窗口监控模块
  - 使用 Win32 API + `psutil` 获取当前前台窗口的 `(process_name, window_title, pid)`。
  - 全程防御性异常处理，任何 Win32/进程异常都不会导致监控线程崩溃。

- `blocker.py`：多层级阻断引擎
  - 从 `config.json` 预构建：
    - `process_blacklist` → 小写 `set`（O(1) 进程级匹配）。
    - `title_blacklist` → 预编译正则列表（窗口标题 / 浏览器标题 / URL 匹配）。
  - 核心方法 `enforce_rules(...)` 实现三道安全闸：
    1. **进程黑名单强力狙杀**：命中 `process_blacklist` 时优先按 PID 精准 `kill()`，失败再按名遍历杀掉所有同名进程，同时通过系统模态弹窗提示。
    2. **浏览器 + LLM 智能护航**：
       - 当前前台为浏览器进程、且存在 `focus_target` 时，静态标题规则在浏览器中完全失效，**一切网页意图改由 LLM 裁决**。
       - 利用 `browser_extension` + `server.py` 上报的 `browser_title/browser_url`，调用 `llm_classifier.evaluate_webpage_intent(...)`：
         - 结果为 `BLOCK` → 通过模拟 `Ctrl+W` 软关闭当前标签页，并弹出 `[AI 护航拦截] 偏离目标：xxx` 的系统原生提示。
         - 结果为 `ALLOW` 或 LLM 异常 → 直接放行，绝不回退到静态规则。
       - 内置 **入口宽限期 + SPA 标题加载宽限期**，避免浏览器首页和单页应用在标题尚未渲染时被误杀。
    3. **静态规则兜底**：仅在非浏览器场景或无专注目标时，根据窗口标题与浏览器标题/URL 走传统正则匹配，命中后通过 `Ctrl+W` + 阻断弹窗进行“温和拦截”。

- `llm_classifier.py`：大模型意图评判引擎
  - 使用标准库 `urllib.request` 调用 SiliconFlow 托管的 `deepseek-ai/DeepSeek-V3` 模型。
  - 构造极其严苛的 System Prompt，只允许输出 `"ALLOW"` 或 `"BLOCK"`，并加入：
    - **特权豁免规则**：对搜索引擎首页（Google/百度）和内容网站首页/搜索页（B 站首页、知乎首页、GitHub 主页等）一律返回 `ALLOW`，保证用户能正常通过这些入口搜索学习资料。
  - 本地 `_DECISION_CACHE` 以“专注目标 | URL”为键缓存判定结果，避免 1.5 秒监控循环频繁触发云端请求。
  - 显式 15 秒超时与异常兜底：任何网络错误或解析异常都打印日志并默认放行，不阻塞主循环。

#### 2.2 现代化 UI 中心（可视化控制面板与高阻力交互）

- `dashboard_ui.py`：FocusGuard 控制中心
  - 使用 CustomTkinter 构建深色主题主窗口，包含三个 Tab：
    1. **专注台 (Focus)**：
       - 输入专注目标（目标文本）。
       - 提供 25/45/60 分钟与 1 分钟测试按钮。
       - 点击时：
         - 将 `时间戳 | 目标 | 时长` 追加写入 `focus_log.txt`。
         - 将 `{"minutes": X, "target": "..."}` 写入 `focus_command.json`，由主进程消费后启动专注。
    2. **规则库 (Blacklist)**：
       - 左侧进程黑名单：展示、增删改 `process_blacklist`，内置：
         - “扫描已安装软件”（解析开始菜单 `.lnk` 得到底层 `exe`）。
         - “一键嗅探活跃进程”（基于 Win32 + `psutil` 抓取当前可见窗口进程）。
       - 右侧标题黑名单：展示、增删改 `title_blacklist`，并提供：
         - “嗅探近期浏览器标签页”：通过本地 HTTP 服务拉取 `recent_tabs`，一键提取标题或 URL 作为规则素材。
    3. **统计局 (Statistics)**：
       - 解析 `focus_log.txt`，计算并展示：
         - 今日总专注时长。
         - 累计总专注时长。
       - 提供按时间倒序的历史专注记录列表，并支持一键刷新。

- `tray_app.py`：托盘 UI 与生命周期总控
  - 使用 `pystray` 创建托盘图标（红色代表专注中，灰色代表闲置）。
  - 菜单包含：
    - 【显示控制中心】：非专注状态下拉起 `dashboard_ui.py` 子进程；专注期间完全禁用，防作弊。
    - 【显示倒计时】：在专注期间重新拉起 `timer_widget.py` 悬浮计时器。
    - 【紧急退出 (需冷静)】：在专注期间打开 `emergency_ui.py`，执行 60 秒冷静期退出流程。
    - 【专注中（不可退出）】：仅作状态展示，不可点击。
    - 【彻底退出】：只在非专注状态下可用，退出托盘并结束后台线程与控制中心子进程。

- `timer_widget.py`：悬浮倒计时小窗
  - 以独立子进程运行，接收专注结束时间戳 `end_time`。
  - 无边框、置顶、略带透明度的小窗口，支持：
    - 实时显示剩余时间（`MM:SS`）。
    - 拖拽到屏幕任意位置。
    - 一键隐藏（仅销毁 UI 进程，不影响专注状态）。
    - 剩余时间 < 60 秒时自动高亮为红色。

- `emergency_ui.py`：紧急安全阀
  - 以独立子进程运行的置顶窗口，强制 60 秒倒计时。
  - 倒计时未结束前，确认退出按钮禁用，只能选择“放弃退出”或关闭窗口（均返回码 1）。
  - 倒计时耗尽后，用户可点击“确认退出专注”，子进程以退出码 0 结束，主进程调用 `state.emergency_stop()` 解除专注。

#### 2.3 端云协同体系（浏览器扩展 + 本地服务 + LLM）

- `browser_extension/manifest.json` + `background.js`
  - Manifest V3 扩展，仅依赖 `tabs` 权限与 `http://127.0.0.1/*` 等本地访问权限。
  - 后台脚本：
    - 监听 `chrome.tabs.onActivated` 与 `chrome.tabs.onUpdated`。
    - 精准捕获当前活动标签页的 `tab.title` 与 `tab.url`，过滤掉 `chrome://`、`edge://` 等内部页面。
    - 将 `{title, url, timestamp}` 通过 HTTP POST 上报到 `http://127.0.0.1:11235/api/tab_update`。
    - 本地服务不可达时静默失败，避免在浏览器控制台疯狂报错。

- `server.py`：本地 HTTP 微服务（内存总线）
  - 使用标准库 `http.server` 实现：
    - `POST /api/tab_update`：接收扩展上报的标签页特征，去重后写入内存列表 `RECENT_TABS`，并更新 `CURRENT_BROWSER_TAB`。
    - `GET /api/recent_tabs`：返回最近的标签页列表，供 `dashboard_ui.py` 的“规则库”可视化使用。
  - 所有共享状态由 `threading.Lock` 保护，保证多线程读写安全。

### 3. 核心功能详述（当前已实现能力）

#### 3.1 事前承诺番茄钟：时长设定与严格锁定

- 用户通过 `dashboard_ui.py` 的“专注台”提前设定本轮专注：
  - 选择时长（25/45/60 分钟或 1 分钟测试）。
  - 输入专注目标（如“复习算法 / 高级数据库报告”）。
- 控制中心将专注信息写入：
  - `focus_log.txt`：记录历史专注日志，供“统计局”分析。
  - `focus_command.json`：以 `{"minutes": X, "target": "..."}` 的格式传递给主进程。
- 主进程读取指令后：
  - 更新 `FocusState.focus_target` 和 `end_time`。
  - 启动悬浮倒计时子进程 `timer_widget.py`。
  - 在整个专注周期内：
    - 禁用托盘的【彻底退出】与【显示控制中心】入口。
    - 仅保留带 60 秒冷静期的紧急安全阀 `emergency_ui.py`。

#### 3.2 多维度拦截防御体系

1. **进程级强力秒杀**：
   - 针对游戏、聊天工具等本地应用，通过 `process_blacklist` 精确匹配进程名。
   - 一旦命中，即按 PID 精准 `kill()`，失败再按进程名遍历杀掉所有同名进程。
   - 同时弹出系统模态阻断提示，确保用户感知明确。

2. **静态标题兜底规则**：
   - 面向所有非浏览器场景（以及无专注目标的情况），通过 `title_blacklist` 的正则匹配窗口标题。
   - 在浏览器拓展不可用、或 LLM 链路断网时，仍然可以基于标题关键字做基础拦截。

3. **AI 护航软阻断（浏览器专属）**：
   - 当前台为浏览器且处于专注模式时：
     - 禁用静态标题规则对浏览器的影响，一切网页拦截改由 LLM 进行语义裁决。
     - 命中 `BLOCK` 时：
       - 模拟 `Ctrl+W` 关闭当前标签页。
       - 弹出 `[AI 护航拦截] 偏离目标：xxx` 的系统模态提示，清晰呈现被拦截的是哪个页面。
     - 命中 `ALLOW` 或 LLM 链路异常时，直接放行，不再继续下探到静态规则。

#### 3.3 LLM 智能护航亮点：DeepSeek 驱动的动态意图识别

- **浏览器特征抓取**：
  - 扩展在标签切换与加载完成时上报 `{title, url, timestamp}`。
  - 本地服务在内存中维护最近标签页列表与当前活动标签页。
  - 阻断引擎在每轮轮询中，将浏览器进程的 `window_title` 与 `CURRENT_BROWSER_TAB` 的 `title/url` 联合使用。

- **缓存防刷与容错策略**：
  - `_DECISION_CACHE` 缓存 LLM 对 `(focus_target, page_url)` 的决策，实现同一网页 0 延迟复用。
  - 云端请求显式 15 秒超时，所有网络/解析异常均默认放行，保证本地循环稳定。

- **SPA 宽限期与入口豁免**：
  - **入口宽限期**：浏览器刚打开或跳转时若尚未获取到 URL，本轮直接放行，等待下一轮再决策。
  - **SPA 标题加载宽限**：若当前 `browser_title` 只是 `browser_url` 的子串（典型 SPA 路由过渡期），一律暂缓裁决，打印 `Browser title loading grace period` 日志并放行，待下轮获取到完整中文标题后再由 LLM 判定。
  - **首页/搜索页特权豁免**：System Prompt 明确要求：搜索引擎首页与内容网站首页/搜索页（B 站首页、知乎首页等导航页）一律 `ALLOW`，保证用户有足够自由度搜索与打开学习资源。

### 4. 控制中心能力：规则库与统计局

- **规则库 (Blacklist)**：
  - 进程黑名单：
    - 手动添加/删除进程名。
    - 一键扫描开始菜单快捷方式，逆向解析真实可执行文件名。
    - 一键嗅探当前活跃窗口进程，将其加入黑名单。
  - 标题黑名单：
    - 维护标题关键字/正则列表。
    - 支持从“近期浏览器标签页”中一键提取标题或 URL，降低规则配置门槛。

- **统计局 (Statistics)**：
  - 读取 `focus_log.txt`，生成：
    - 今日专注时长汇总。
    - 累计总专注时长。
    - 历史专注记录明细列表。
  - 支持一键刷新，方便用户在长时间使用后获得持续性正向反馈。

### 5. 当前开发进度与系统成熟度

- **链路打通情况**：
  - 已完成从“控制中心发起专注 → 主进程状态更新与计时器拉起 → 后台监控前台窗口与浏览器标签 → 多层级阻断引擎执行 → LLM 云端裁决”的完整闭环。
  - 浏览器扩展、HTTP 本地服务、LLM 推理引擎三者协同稳定，实测场景下已能：
    - 放行 B 站等平台上的学习类视频/技术内容。
    - 拦截与当前专注目标无关的搞笑、娱乐、纯刷屏内容。

- **工程鲁棒性与可部署性**：
  - 所有关键路径均围绕“后台线程绝不崩溃”“UI 与托盘进程解耦”设计，已大量覆盖异常兜底与冷却机制。
  - 阶段九完成后，FocusGuard 已具备在个人环境长期驻守的实机部署能力：
    - 支持长时间运行、反复启停专注与紧急退出。
    - 支持在浏览器中长时间刷选学习资源，而不会被静态规则误杀入口。

- **下一步可演进方向（非必需，仅展望）**：
  - 在统计局中增加“按专注目标聚合”的分析视图。
  - 为 LLM 裁决结果增加可视化调试面板（例如展示最近一次网页判定的上下文与结果）。
  - 扩展更多浏览器与多设备同步能力（保留在未来版本考虑）。
